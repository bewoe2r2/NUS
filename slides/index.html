Key innovations

What sets Bewo apart.

01

Baum-Welch Personalization

HMM learns per-patient emission parameters from 42+ observations. Mr. Tan's "normal" glucose is 7.5, not the population 6.5.

02

Counterfactual Engine

"Take Metformin → glucose -2 mmol/L." Re-runs Monte Carlo on modified observation vector. Risk reduction is calculated, not guessed.

03

Loss-Aversion Gamification

Prospect Theory: losses weigh 2.5× more than gains. $5/week you fight to KEEP, not earn. QR redemption Sundays.

04

Outcome-Based Tool Selection

14-day exponential decay half-life. Agent tracks which of 18 tools actually improved this patient's state.

05

Cross-Session Agent Memory

Episodic → semantic consolidation. 5 "high glucose after lunch" episodes become 1 pattern: "post-lunch spikes."

06

Proactive Oracle

6 triggers scan continuously: glucose_rising, sustained_risk, logging_gap, med_nudge, streak_save, mood_followup.

🔒

Offline-First HMM

Viterbi, Baum-Welch, Monte Carlo all run locally. No internet for state inference. Only NumPy.

🛡

De-identified AI Calls

_deidentify_profile_for_llm() strips name, NRIC, address, phone, email. Only age + conditions + medications sent.

⏳

3-Tier Data Retention

Identity: local only. Health: encrypted, 6-month auto-purge. Cloud: anonymised outcomes only. PDPA compliant.
